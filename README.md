# What Did I Just Sign?

A high-fidelity HCI prototype that helps people understand privacy policies before they agree to them. Users can paste a privacy policy URL or text, choose an output language, and receive a quick privacy scan with a score, short summary, risk cards, and traceable policy excerpts.

The demo is designed to answer one question quickly: **is this policy privacy-friendly, acceptable, or invasive?**

## What It Does

- Fetching a privacy policy in any language from a URL or accepting pasted text
- Summarizing the policy in plain English, Chinese, Korean, or Norwegian (user selects langauge)
- Scoring how invasive the policy is from `0-100`
- Highlighting important clauses
- Flagging concerns based on user preferences, such as location tracking, third-party sharing, children's data, ads, biometrics, or data retention
=======
- Accepts a privacy policy URL or pasted policy text.
- Sends the policy to a FastAPI backend for AI analysis.
- Returns a compact, mobile-style privacy scan.
- Shows an overall policy score from `0-100`.
- Breaks risk into glanceable cards: data collection, third-party sharing, retention, and tracking.
- Opens modal details when a risk card is clicked.
- Shows original policy evidence only when the user asks for traceability.
- Supports English, Norwegian, Chinese, and Korean UI/output language flows.

## How The Policy Score Works

The score is a `0-100` privacy score where **higher is better**:

- `80-100`: Privacy-Friendly
- `60-79`: Acceptable
- `40-59`: Moderately Invasive
- `0-39`: Highly Invasive

The backend asks Qwen to evaluate the policy using a privacy rubric. It weighs:

- how much data is collected
- whether data is shared with third parties
- tracking, cookies, ads, or analytics practices
- data retention and deletion clarity
- user rights such as access, correction, deletion, and opt-out choices

There is no fixed arithmetic formula like `score = A + B - C`. The score is generated by the AI model from the policy text and rubric. The app then applies exact frontend logic to display the score:

```js
if (score >= 80) return "Privacy-Friendly"
if (score >= 60) return "Acceptable"
if (score >= 40) return "Moderately Invasive"
return "Highly Invasive"
```

The circular score graphic also uses the exact percentage:

```js
progress = score / 100
```

## Run Locally

Start the backend:

```bash
cd backend
source venv/bin/activate
uvicorn main:app --reload --port 8000
```

Start the frontend in a second terminal:

```bash
cd frontend
npm install
npm run dev
```

Frontend environment:

```env
VITE_API_URL=http://localhost:8000
```

Backend environment:

```env
QWEN_API_KEY=your-dashscope-key
QWEN_API_BASE=https://dashscope-intl.aliyuncs.com/compatible-mode/v1
```

## Current Demo Status

The prototype is ready for the demo. The backend analyzes real or pasted policies, and the frontend presents the results as a lightweight privacy health check rather than a dense legal report.

The design intentionally uses progressive disclosure: the first screen gives the user the main takeaway, and deeper legal evidence appears only when the user clicks into a risk card.
